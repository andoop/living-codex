# 诚实模型 · The Honesty Model

Living Codex 的全部价值压在一句话上：**你敢信它写的每一句话。** 本文讲它怎么做到。

## 三档可信度

| 档 | 含义 | 什么时候能用 |
|---|---|---|
| `已确认` | 被引用的符号在文件里**确实出现过**，并已用 grep 核对。**仅此而已**——不代表它的行为、关系是对的。 | 只用于静态结构结论，且有 grep 回查能力时 |
| `推断` | 没跑过、靠读代码得出的判断。一切行为、关系、顺序、因果、运行期结论都归这一档。 | 任何时候 |
| `未解之谜` | 得跑起来、需要更多上下文、或没读到才能定。 | 任何时候 |

## 五条死规矩

1. **「已确认」只担保字面存在。** grep 到一个名字，只证明这个名字出现过，不证明围绕它写的那句话是对的。
2. **行为类结论封顶到「推断」。** 调用、顺序、触发、回滚、重试、并发、校验、持久化……这些得跑代码、或要读好几处控制流才能确定的，**再有把握也只标「推断」**（哪怕开了解析器后端也一样）。
3. **grep 回查是必需，不是可选。** 没有 `verify-citations.sh` 或等价能力时，「已确认」这一档**直接停用**，全部降成「推断」。
4. **解析器后端只升级静态结构。** 开了 `codegraph` 时，import、定义、调用点这类**静态结构**可以升到「已确认(解析器)」；但**运行期行为永远是「推断」**——对一个调用，只敢说"这里写了一个调用"，不敢说"运行时真会走到这"。
5. **永不假装验证。** 独立审核只做形式和结构上的核对，结论是"没被挑出毛病"，**不是**"运行时已验证"——绝不出现"已验证 / 运行时成立"这种话；发现两处叙事打架，只标出来提醒，**绝不擅自改一边制造虚假一致**。

## 机械校验只证三件事

跑通校验脚本（`ledger-orphans.sh` 退出码 0），只证明这三类**清单列全了**：

- 文件清单全（每个该算的文件都被某条业务认领，或明确标为"无主"）；
- 入口清单全（每个能 grep 到的框架入口都被列出或明确排除）；
- 连边清单全（每个能 grep 到的异步/派发起点都配了对端或标了"断裂"）。

**它绝不等于**"业务没遗漏"或"已验证正确"。所以覆盖率一律叫"枚举覆盖率"，不叫"业务覆盖率"。而且入口、连边、状态这三处**没有机械底数**（grep 搜不全的部分就是公认的盲区），首页会明确点出来。

## 成本与覆盖上的诚实

- 用 `--journey` 把任务切小，一次只追一条旅程；每条旅程都有 token 预算。
- 预算超了、只能粗读的旅程，会**显眼地标上"部分追踪"**，没读到的进"未解之谜"，绝不偷偷标成"已确认"。

## 安全

- 密钥、凭证、个人信息：只记键名、位置、用途，**绝不写原文**（中间草稿和日志也一样）。零运行时下这是"尽力"而非"保证"，已如实声明，并提供可选的 `scrub-secrets.sh` 兜底体检。
- 全程只读：除了产物目录（`docs/codebook/narrative/`）和账本文件，**不动目标项目的任何源码**。
