# 诚实模型 · The Honesty Model

Living Codex 的全部价值压在一句话上：**你敢信它写的每一句话。** 本文定义它如何做到。

## 三档可信度（confidence）

| 档 | 含义 | 何时可用 |
|---|---|---|
| `已确认` | **仅**：被引符号/字符串在所标文件里**字面存在**，且已被 `verify-citations.sh` 机械 grep 核验。**不**担保语义/行为/关系为真。 | 仅静态结构结论 + 有 grep 回查能力 |
| `推断` | 未经运行验证的判断（含一切行为/关系/顺序/因果/运行期/动态结论）。 | 任何时候 |
| `未解之谜` | 需运行、需更多上下文、或未读到才能定。 | 任何时候 |

## 五条铁律

1. **「已确认」只担保字面存在性。** grep 命中一个符号名，只证明"这个名字出现过"，不证明围绕它写的那句话为真。
2. **行为结论封顶「推断」。** "调用 / 在…之前 / 触发 / 回滚 / 不重试 / 并发 / 校验 / 持久化"等——凡需运行、或需读多处控制流才成立的谓语，**在任何档位（含解析器后端档）最高只能「推断」**。
3. **grep 回查是必需，不是可选。** 宿主无 `verify-citations.sh` 或等价能力时，「已确认」档**整体不可用**、全部降级「推断」。MUST 不挂在可选项上。
4. **解析器后端只升级静态结构。** `--backend codegraph/repomix` 在场时，import/定义/调用点语法存在等**静态结构**可升「已确认(解析器)」；**运行期/动态行为永远是「推断」**。对调用只担保"调用点语法存在"，被调具体实现（多态/接口/DI/反射）一律推断。
5. **永不假装验证。** 独立 AUDIT 只做形式/结构/枚举类核查，结果是"未被独立复读证伪"，**不是**"运行时已验证"——绝不出现"已验证 / 运行时成立"等措辞；发现叙事间矛盾只标 advisory，**绝不擅自改一边制造虚假一致**。机械闸门（`ledger-orphans.sh` exit 0）只证三类「可 grep 范围内」枚举完整，**绝不**等于"业务零遗漏/已验证正确"。

## 双档承诺

- `backend=none`（纯模式）= **最佳努力 onboarding 草稿**：跨边、subsystem 分组、运行期结论一律「推断」。
- 解析器后端在场 = 静态结构可标「已确认(解析器)」。

承诺随档位同步升降。confidence 图例**逐章首显著放出**，免责不只在书首页一处——避免局部「已确认」图章盖过全局声明。

## 成本与覆盖诚实

- 用 `--journey <名>` 把任务切小（只追一条旅程）；每条 journey/角色设 token 预算。
- 超预算降级为摘要级的旅程**必须显著标「部分追踪」**，未读文件/目录入「未解之谜」，confidence 不得标已确认。
- **覆盖率冠"枚举"二字**：`枚举覆盖率`（可 grep 范围内三账本枚举完整度）≠ "业务覆盖率"；入口/连边/状态三处分母**无机械底线 = 承认盲区**，首页须显式点破。

## 安全

- 密钥/凭证/PII：只记键名/位置/用途，**不写原文**（含中间草稿与日志）。零运行时下这是**尽力而非保证**，已如实声明；提供可选 `scrub-secrets.sh` 兜底。
- 只读：除约定输出目录（`docs/codebook/narrative/`、账本文件）外，**不修改目标项目任何源码**。

> 这套纪律继承自 [Sandtable](https://github.com/andoop/sandtable) 的「实事求是、不猜测、不捏造」四底线。Living Codex 是它在「代码理解」上的落地。
