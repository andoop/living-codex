# references/self-review.md — 二次自审（诚实化）

> 落实 FR6。对应 TC3。这是「原自我红蓝对抗」的诚实化版本——**自审 ≠ 验证**。

---

## 自审能做什么 / 不能做什么

| 能（确定性硬查） | 不能 |
|---|---|
| 被引符号/文件**是否存在**（grep 回查） | 判定结论行为**是否为真**（需运行） |
| `[[wikilink]]` 是否失效 | 验证调用关系/执行顺序 |
| 孤儿页（无人链接） | 消解「需运行才知」的歧义 |
| 文本内部是否自相矛盾（**仅 advisory**） | 给「安全感」保证 |

---

## 硬查项（参与「通过」判定）

1. **被引符号/文件存在性**：对每条 `已确认`，跑 grep 回查（`scripts/verify-citations.sh`）。未命中 → 降级 `推断` + 加注。
2. **失效 wikilink**：`[[X]]` 指向不存在的页 → 报告并修链接（不改结论语义）。
3. **孤儿页**：无任何页链接到它 → 报告（提示补链接或归并）。

## Advisory 项（**不**参与通过判定）

- **文本内部自相矛盾**：A 章说「用 Redis」、B 章说「用 Memcached」→ **只提示**，**不擅自挑一边改写**（无运行时真值不能判定谁对）。在 99-未解之谜记录「存在矛盾，待人工/运行确认」。

---

## 硬声明（自审产物必含，逐字精神）

> 本次自审 = 「**未被二次复读证伪**」，**不是「运行时已验证」**。
> 本产物**不含**「已验证 / 已确认运行行为」措辞。
> 需运行才能确认的结论一律在「未解之谜 / 推断」。

## 禁用措辞清单（出现即自审不通过）
`已验证` / `已确认运行行为` / `运行时成立` / `保证` / `验证通过`（用于行为结论时）。

> 注：`已确认` 仅可用于「字面存在性」语境，且必须 grep 回查过；`已确认(解析器)` 仅静态结构。两者均**不可**修饰行为结论。

## 自审流程
1. 收集全书所有 `已确认` 标注 → 逐条 grep 回查 → 降级未命中者。
2. 扫描全部 `[[wikilink]]` → 报失效。
3. 构建链接图 → 报孤儿页。
4. 文本矛盾扫描 → 写入 advisory 清单（不改）。
5. 措辞扫描 → 命中禁用清单则**不通过**，回 PRESENT 修措辞。
6. 输出自审报告（含硬声明）。
