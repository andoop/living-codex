# references/blind-spots.md — 盲区与缺口前置具名（FR14 · A-RT3-6）

> 落实 PRD FR14。验证 TC24。
> 核心：**禁止用单一高"枚举覆盖率"数字粉饰盲区子系统**（95% 会淹没"整个子系统 0 覆盖"）。
> **与每文档 gap 小节的分工**（见 `plain-language.md` §6 / `narrative-book-spec.md`）：每条 journey/rule 与 L0 各有白话 `## 还不知道什么`（装该处"未解之谜"）；本专章是**全局盲区汇总**（识别到存在但本轮未追踪的子系统），二者互补、均线上可见，不互相替代。本专章带 `type: blind-spots` frontmatter。

---

## 禁单一覆盖率粉饰

- **禁止**只给单一高"枚举覆盖率"数字。
- 覆盖率必须**分轴 / 分子系统呈现** + 就地标注 **"未知非高、可能存在未覆盖子系统"**。
  - 例：
    ```
    | 子系统 | code 枚举覆盖率 | entry 枚举覆盖率 | edge 配对率 | 备注 |
    |---|---|---|---|---|
    | 上传链路 | 92% | 8/8 | 5/6 | 1 断裂边 |
    | 推送子系统 | 0% | 0/3 | — | ⚠️ 本轮未追踪，见缺失专章 |
    ```

---

## 强制"已知缺失/未测子系统"专章

主动扫框架签名（DI 模块、SPI、反射注册、自定义事件总线）列出"**识别到存在但本轮未追踪**"的子系统——**具名而非沉默**。

- 该章**不得被首页免责声明一笔带过**。
- 扫描签名示例：`@Module` / `META-INF/services` / `Class.forName` / 自定义 Bus 类名。

## 章首必标 caveat（RT4-P2a · 非盲区全集）

> **本章仅含可识别签名的缺失子系统，不构成盲区全集；未知未知（unknown-unknown）框架不会在此出现。**

- 避免读者误读为"盲区已尽列"。
- 与入口分母（`entry-ledger.md`）、连边分母（`edge-ledger.md`）、状态轴（`state-axis.md`）的非对称盲区声明一致，统一口径见 `honesty-charter.md`。
