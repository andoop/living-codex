# references/provenance.md — 诚实 provenance（核心 · v2.1 收紧）

> 落实 SKILL.md 诚实红线（confidence 三档 / 行为性结论封顶）。对应 FR5/FR8。**本文件是全 skill 最高红线所在。**

---

## confidence 三档（唯一合法取值）

| 档 | 含义 | 何时可用 |
|---|---|---|
| `已确认` | **仅**：被引符号/字符串的**字面存在性**已被机械 grep 核验。**不**担保语义/行为/关系为真。 | 仅静态结构结论 + 有 grep 回查能力 |
| `推断` | 基于阅读得出的判断，未经运行验证。**所有行为性结论的上限。** | 默认档 |
| `未解之谜` | 不知道 / 需运行才能定 / 未读到。 | 缺证据时 |

> 解析器后端档增加一个**子标**：`已确认(解析器)` —— 仅用于静态结构结论（见下）。

---

## 「已确认」严格定义（不可放宽）

`已确认` = 「我在文件 `X` 里 grep 到了字符串/符号 `S`」。**仅此而已。**
- 它**不**意味着 `S` 的功能、行为、被调用关系、执行顺序为真。
- 它**必须**经过 grep 回查（`scripts/verify-citations.sh` 或等价）。**无回查能力 → 「已确认」档不可用，自动降级「推断」**（MUST，不挂在 optional 上）。

## 行为性结论：任何档位最高「推断」（最高红线）

**判定准则：凡需运行、或需读多处控制流才成立的谓语 = 行为性。**

行为性谓语（**包括但不限于**）：
> 调用 / 在…之前 / 在…之后 / 触发 / 回滚 / 重试 / 不重试 / 并发 / 加锁 / 校验 / 持久化 / 去重 / 缓存 / 降级 / 路由到 / 异步回调 / 影响 / 导致

→ **在任何档位（含解析器后端档）都不得标「已确认」/「已确认(解析器)」，最高只能「推断」。**

❌ 错误：`AuthService 调用 TokenStore.verify()`（confidence: 已确认）
✅ 正确：`AuthService.login 内出现对 TokenStore.verify 的调用点（已确认调用点存在）；其运行期是否实际执行/被何实现处理 = 推断`

## 解析器后端档边界

`backend=codegraph` 时，**只允许把静态结构结论**升级为 `已确认(解析器)`：
- import / 定义存在；
- **调用点语法存在**（只担保「这里有一个对 B 的调用语法」）；
- 静态调用边。

**永远「推断」**：被调具体实现、多态分派、接口/DI/反射目标、运行期实际路径、并发行为。
> 即解析器档也**不得**渲染成「A 调用 B（运行时成立）」。对调用只担保「调用点语法存在」。

## 引用格式（不编行号）

- 格式 = **「文件路径 + 可 grep 的符号名/字符串」**，例：`src/auth/service.py :: class AuthService` / `src/auth/service.py :: "TOKEN_TTL"`。
- **不强行标行号**（LLM 行号不可信）；不编造 `file:line`。
- 行号可作为「大致位置」附注，但**不得**作为「已确认」的依据。

## confidence 图例逐章首（FR8）

每个 journey/rule 文件**逐节首**必须放出图例（见 `templates/narrative-journey.md` / `templates/narrative-rule.md`），含释义：
> `已确认` = 仅字面存在性、已 grep 核验，**非语义/运行验证**；`推断` = 未经运行验证的判断；`未解之谜` = 需运行或未读到。

免责不止书首页一处——避免局部「已确认」图章盖过全局声明。

## grep 回查协议
1. PRESENT/review 时，对每条 `已确认` 结论，取其「文件 + 符号」喂给 `scripts/verify-citations.sh`。
2. grep 命中 → 保留 `已确认`；grep 未命中 → **降级为 `推断`** 并在该条加注「回查未命中」。
3. 宿主无脚本/无等价 grep → 全库 `已确认` 档停用。
