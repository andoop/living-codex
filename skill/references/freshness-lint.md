# references/freshness-lint.md — 叙事层保鲜 / staleness（FR11 · B3）

> 落实 PRD FR11。验证 TC20。
> 核心诚实声明：**grep 命中符号 ≠ 叙事未过期**（符号在、语义可能已变）。保鲜是 **advisory 标记**，不是"已保鲜"保证。

---

## `narrate --lint` 反向索引

```
输入：coverage-ledger 的"符号 → 业务节点"映射
构建：反向 file → journey 索引（一个文件被哪些 journey/rule 消费）
触发：git diff 命中的文件
动作：把消费该文件证据的所有 journey/rule 标"⚠️ 待复核（可能过期）"
```

- 索引来源 = 覆盖账本（已登记的符号→节点映射），不另建数据源。
- 标记是 advisory：提示"这条叙事的证据文件变了，请人复核"，**不自动判定过期或自动改写**。

## 为什么只能 advisory（诚实边界）

仅靠 grep 命中符号**不能**证明叙事未过期：
- 符号仍在，但语义可能已变（如"重试 3 次"代码改成 5 次，符号 `RETRY_MAX` 还在）。
- 故 `--lint` **只能标"证据文件已变、待复核"**，**不能**标"已确认未过期"。

## 降级（无 git / 无脚本）

产物首页显式声明快照：
```
> 本书为 <commit-hash / 时点 2026-06-23> 的快照。
> 代码若已变更，结论可能失效；请以当前代码为准复核。
```

- 无 git → 无法算 diff → 只能首页快照声明，不提供 file→journey 反向触发。
- 须在首页明写"本轮无 staleness 自动检测，依赖快照时点"。
