# references/personas.md — 4 人设视角（禁同质化）

> 每个人设**绑定不同入口章节 + 不同关注维度 + 不同风险清单**。
> 红线：视角章节内容必须**互不相同**——不是换个标题重复同一段。每人设有独立「先读什么」与「最怕什么」。

---

## P1 · 架构师 architect
- **入口章节**：D1 架构总览 → D2 模块边界 → D3 数据流（标推断）。
- **关注**：分层是否清晰、模块耦合、跨边界依赖、可扩展点。
- **风险清单**：循环依赖、上帝模块、跨层穿透调用、隐式全局状态。
- **不关心**：单元测试细节、具体配置项默认值。

## P2 · 新人 newgrad
- **入口章节**：D5 构建与运行（先跑起来）→ D2 模块（核心几个）→ D6 测试（怎么验证改动）。
- **关注**：如何 build/run、最小可改动闭环、术语表（concepts）。
- **风险清单**：环境搭建坑、易踩的隐式约定、缺文档的「祖传」逻辑。
- **不关心**：全局架构权衡、SRE 容量规划。
- **专属产物**：onboarding 依赖序路线（见 `presentations.md`）。

## P3 · 安全 security
- **入口章节**：D7 接口/API → D9 配置 → D11 风险 → D8 数据模型。
- **关注**：信任边界、输入校验点（**校验=行为性→推断**）、鉴权位置、密钥/凭证处理。
- **风险清单**：未校验输入入口、硬编码凭证位置（**只记键名/位置，不记原文**）、过宽权限、注入面。
- **不关心**：构建优化、代码风格。

## P4 · SRE
- **入口章节**：D5 构建与运行/部署 → D9 配置 → D10 历史热点 → D11 风险。
- **关注**：部署拓扑、可观测性（日志/指标位置）、故障域、配置外部化。
- **风险清单**：单点、缺重试/超时（**行为性→推断**）、配置漂移、缺监控埋点。
- **不关心**：API 设计美学、新人术语表。
